# 小數獨 設計 Playbook（推理系列首款）

> 2026-07-17 經 codex-review 兩輪真共識確立（sudoku-design-quality）。本檔＝小數獨/推理系列設計工作的專案層 SSOT；通用設計鐵則在 `.claude/agents/designer.md`；系列視覺語言只在 Lori 拍板後回填品牌中樞 §2-C。

## 0. 設計 Brief（一頁）

- **產品**：小數獨——認真推理訓練的兒童數獨。玩永遠免費零廣告；教練層（引導觀察、永不代填）一次性買斷。
- **目標使用者**：小學階段的孩子（⏳ 精確年齡帶待 Lori 確認）＋掏錢的家長。家長要一眼看出「方法論清楚、看得見成長」。
- **裝置**：iPhone（直向為主）、iPad（直橫）；PWA 與 SwiftUI 原生雙軌，同一套視覺語言。
- **品牌禁區**（品牌中樞 §1/§2-C）：不幼稚化、不搞笑、零廣告、零追蹤、無黑暗誘導；不販賣焦慮。
- **成功條件**：Lori 認得出「這是用心設計的」；5 秒盲測六題全過；兒童測試過三級門檻。

## 1. 執行十步（共識定序，不可跳）

1. Brief（本檔 §0）
2. 標竿研究（§2，半天上限）
3. 資訊層級＋狀態矩陣（§3）
4. **只做兩個真正不同的方向**，每個用相同內容展示 4 個關鍵狀態（§4）
5. Lori 選方向（可逆條款：兒童測試若顯示互動看不懂，允許回修該方向的互動表現，不重開風格探索）
6. 選定後才抽 token／字型／元件規格（§5–6）
7. 「玩一局＋一個 L2 教學框」真機 demo
8. 兒童／家長測試（§7 三級門檻）
9. 自批＋跨模型驗證＋裝置 QA（§8 分級）
10. 通過後才全面套 PWA 與原生版

## 2. 標竿研究規則

- 首輪上限：**4–6 款、每款 4–6 個關鍵狀態、半天結束**。三類必備：成熟謎題產品（Good Sudoku／Puzzmo／LinkedIn Queens/Tango）＋優秀兒童學習/工具產品＋Apple 遊戲與無障礙 HIG。
- **逐狀態研究**，不只首頁：初始盤面／選格與同行列宮提示／輸入／筆記／錯誤衝突／復原／教學提示／完成一局／付費入口／深色與 iPad。
- 招式記「**關係、目的、適用場景**」不記絕對像素（例：「宮界約為格線 3–4 倍，層級清楚不搶數字」）。
- 每條招式帶：來源＋日期＋平台＋畫面狀態｜解決什麼問題｜適不適合小數獨｜**採用／待試／拒絕＋理由**｜品牌風險。
- ⚠️ **repo 是 public**：benchmarks 只存來源連結＋文字分析＋必要小裁圖；不放整套競品截圖；成品避免貼近單一標竿的整體識別。

## 3. 狀態矩陣（小數獨全狀態，設計與 QA 共用）

初始盤面｜選格（含同行列宮與同數字高亮）｜輸入數字｜筆記模式（候選小字）｜錯誤與衝突｜復原/擦除｜提示 L1 觀察／L2 技巧／L3 示範｜完成一局｜技巧圖鑑與貼紙（收集，全本機）｜付費教練入口（家長門檻感、無焦慮話術）｜接續上次入口｜深色模式｜iPad 直橫。

## 4. 方向盤規格（重做版）

- 兩個方向，**彼此真正不同**；每方向同樣呈現 4 個關鍵狀態：選格中的盤面／筆記＋一個錯誤格／L2 教學提示／完成畫面。
- 每方向至少 **3 項有意圖、可驗證的設計決策**——每項說得出解決什麼問題，說不出就移除。紙紋、陰影、徽章是候選手段不是必備。
- 裝置呈現（分級 QA 第一級）：iPhone 淺色全套＋至少一個深色關鍵狀態＋iPad 代表版面。
- 門檻＝**5 秒盲測六題**：這是什麼產品？第一眼該做什麼？哪格被選中？哪裡是錯誤？感覺認真溫暖還是幼稚搞笑？像不像便宜模板？

## 5. Token 規則（方向拍板後才建）

- 語意命名，不散用原始色階：`boardBackground / cellBorder / boxBorder / givenDigit / playerDigit / candidateDigit / selectedCell / relatedCell / conflictCell / focusRing / coachHighlight`。
- Radix Colors 當色彩骨架（12 階、深淺邏輯），**每個實際配對仍逐一量測對比**（一般文字 ≥4.5:1；重要 UI 與狀態的非文字對比 ≥3:1；顏色不得單獨傳達錯誤/選中）。
- 一份小型平台中立 token 表（JSON/MD）＝單一真相 → 手動映射 CSS variables 與 Swift Color/Asset Catalog（不建生成器）。
- 字級以「角色＋可讀性」定，modular scale 與 8pt grid 僅參考；盤面幾何獨立處理（可用 4pt 子格與奇偶像素修正）。

## 6. 字型規則

- 允許內嵌 **OFL 開源字型**（iOS bundle＋UIAppFonts；PWA self-host subset），零外部 CDN 不變；中文 UI 維持系統字。
- 逐款驗證授權、保留 license 檔、留意 Reserved Font Name；PWA 只載必要字重與拉丁數字 subset；iOS 與 PWA 同一版本來源；Fontsource 只當取檔來源、不進 runtime。
- 數字驗收：1/7、3/8、5/6/9 不易混｜lining＋tabular figures｜given/player/candidate 三層清楚｜小尺寸候選數不糊｜深色不膨脹｜字型載入失敗 fallback 不破版。

## 7. 使用者測試三級門檻

| 階段 | 誰 | 擋什麼 |
|---|---|---|
| 方向選擇 | Lori | 不需等兒童測試 |
| Demo 迭代 | **佑佑（目標年齡，必測）**：不提示完成選格/填數/筆記/復原/叫教練，觀察卡點不問喜好 | 發現主要理解與操作問題 |
| **上架送審前（硬門檻）** | **外部第二位兒童**：不同家庭、第一次接觸、無人提示、年齡帶另一端 | 單一兒童樣本不足以放行上架 |

家長測試由 Lori 承擔（理解免費/付費邊界與教練價值）；她是產品決策者，能驗品牌與購買邏輯、不能當獨立市場證據——外部家長列建議不列門檻。

## 8. QA 分級（裝置分級，狀態不延後）

- **方向盤**：4 關鍵狀態 × iPhone 淺＋深色關鍵態＋iPad 代表面。
- **Demo**：加實機操作、放大文字、離線、字型 fallback。
- **上架前（硬門檻）**：完整 裝置 × 外觀 × 無障礙（放大字/高對比/VoiceOver）× 全狀態矩陣。
- 每輪自批分責並附「發現→修改→證據」：R1 視覺層級/構圖/品牌/原創性；R2 互動狀態/可讀性/無障礙/跨裝置。跨模型驗證輸出結構化（通過否/嚴重度/違反條目/指向元素/是否阻擋/最小修法）。

## 9. 外部資源安全（合身版）

能複製少量靜態 token 就不裝套件｜鎖定精確版本與 commit｜不執行第三方 build/install scripts｜外來 SVG 查 script/外連/event handler/foreignObject｜字型與 CSS 全本機打包｜檢查 build 產物無外部連線｜有 npm 依賴才跑 audit。審「實際帶進產品的那個版本與檔案」，不對單一色票 repo 做全量原始碼審查。
